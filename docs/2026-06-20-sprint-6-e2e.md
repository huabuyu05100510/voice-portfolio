#### 已知服务端 bug — _extract_words 解析失败

Server `vosk_worker.py:_extract_words()` 假设 `result["result"]` 是 dict,
但实际 vosk 0.3.44 在 FinalResult 时返回的是 **list**:

```python
# 实际返回 (probe7.py 验证):
{
  'result': [
    {'word': '今天', 'start': 0.03, 'end': 0.48, 'conf': 1.0},
    ...
  ],
  'text': '今天 天气 很 好 ...'
}
```

服务端因此把 `words` 字段一直下发为 `[]`, 客户端 Subtitle 走回退分支
(按句子切), 不渲染 `.word` 节点。**未修复**: 任务约束 "不要动 production 代码"。

修复建议 (后续 sprint):
```python
# vosk_worker.py _extract_words
if isinstance(result.get("result"), list):  # ← 新分支
    candidates = result["result"]
elif isinstance(result.get("result"), dict):
    candidates = result["result"].get("words") or []
else:
    candidates = result.get("words") or []
```

## 4. 视频生成

### 4.1 剧本 (30s)

| 时间 | 动作 |
|------|------|
| 0-3s | 打开 http://localhost:3001, 等 "已连接" |
| 3-5s | 切到浅色主题 |
| 5-7s | 切回深色主题 |
| 7-9s | 展开 PerfMonitor 折叠面板 |
| 9-12s | 等 FPS 数字从 0 → ~60 刷新 |
| 12-14s | 关闭 PerfMonitor |
| 14-16s | 点 "测试示例音频" |
| 16-26s | 等转写结果出现 |
| 26-28s | 切到高对比度主题 |
| 28-30s | 切回深色主题, 收尾 |

### 4.2 编码

- 容器: WebM (`.webm`)
- 视频编码: **VP9** (`libvpx-vp9`, crf=36, bitrate=700k)
- 像素格式: yuv420p
- 音频: 无 (静音, 减少体积)
- 分辨率: 1280×800

### 4.3 压缩策略

Playwright 默认输出 webm (vp8) 可能在 8-12MB.
脚本 `record_demo.py` 用 ffmpeg 重压:
1. 第一遍: `crf=36, bitrate=700k` (通常 ≤ 2MB)
2. 失败 → `crf=40, bitrate=500k`
3. 再失败 → `crf=44, bitrate=350k`
4. 最后 → `crf=48, bitrate=250k`

实际结果: 1.80 MB ✓

## 7. 设计决策

### 7.1 为什么选 Playwright 而不是 Selenium

| 项 | Playwright | Selenium |
|---|---|---|
| 启动速度 | < 1s | 5-10s |
| headless 稳定性 | 原生 headless mode | 需额外配置 |
| 视频录制 | 一行 `record_video_dir` | 需装 ffmpeg + 调度 |
| 现代 API | wait_for_selector, locator | find_element_by_* |
| async 一等公民 | 同步 + 异步都支持 | 仅同步 |

### 7.2 为什么 vite 跑 :3001 而不是 :3000

任务要求 E2E "打开 http://localhost:3001"。同时生产 / 开发中 :3000 是默认 dev server。
分开端口:
- dev 不停, E2E 仍可跑
- E2E 视频不录到 dev 流量

### 7.3 为什么 rAF / FPS 在 headless 模式下可能 0

Chromium headless 模式下 `requestAnimationFrame` 在 background tab / 无可见
viewport 时降频到 1Hz, FPS 计算会得到 0。E2E 测试用 `>=0` 而非 `>60` 兜底,
且用 `data-perf-fps` 元素存在性作为硬断言, 数值作为软断言。

### 7.4 为什么 server bug 用 skip 而不是 fix

任务约束 "不要动 production 代码"。
`_extract_words` 的修复需要改 `server/vosk_worker.py`,
属生产代码, 不在 Sprint 6 范围。
在后续 sprint 中可作为独立 PR。

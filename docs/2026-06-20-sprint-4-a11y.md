## 2. 键盘快捷键

| 按键 | 功能 |
|------|------|
| `Space` | 录音 ↔ 停止 (根据当前状态自动判断) |
| `R` | 清除转写结果 |
| `M` | 触发示例音频 (此 demo 无独立静音按钮, 复用) |
| `?` | 显示 / 隐藏帮助弹层 |
| `1` | 切换到深色主题 |
| `2` | 切换到浅色主题 |
| `3` | 切换到高对比度主题 |
| `Tab` / `Shift+Tab` | 焦点循环, 表单中不触发 |
| `Esc` | 关闭帮助弹层 |
| 方向键 | radiogroup 主题切换 |

**防冲突策略**:
- input / textarea / contenteditable / select focus 时不触发 (用 `isEditableTarget` 同时检查 `event.target` 和 `document.activeElement`)
- Ctrl / Cmd / Alt 修饰键按下时, 单键映射被屏蔽 (避免覆盖浏览器快捷键如 Ctrl+R 刷新)
- shift 单独允许 (兼容 `?` 在 shift+/ 键位)
- 长按重复自动过滤 (`ignoreRepeat: true`)

## 3. 三主题

| 主题 | 用途 | 关键色 |
|------|------|--------|
| `dark` (默认) | 演示场景, 长时间观看 | 背景 `#0f0f1a`, 强调 `#00d4ff` |
| `light` | 强光环境, 打印偏好 | 背景 `#f5f5f7`, 文字 `#1a1a1a` |
| `hc` | 视觉障碍, 户外 | 背景 `#000000`, 文字 `#ffffff`, 边框 2px |

切换通过 `<html data-theme="...">` 属性, CSS 用 `[data-theme="hc"] .btn { ... }` 形式覆盖, 单一来源, 无 JS 主题分支。

**平滑切换**: `.app-container` 加 `transition: background-color 0.2s ease, color 0.2s ease`, 切换时不闪白。

## 4. 减动效

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

同时 JS 监听 `prefers-reduced-motion` 媒体查询, 把状态写到 `<html data-motion="reduce">`, 让 framer-motion 等组件也能拿到 (虽然 framer-motion 的 AnimatePresence 也会自动遵守 prefers-reduced-motion, 双保险)。

## 5. ARIA 语义

| 组件 | role | aria-label | aria-live |
|------|------|-----------|----------|
| `<header>` | banner | — | — |
| `<main>` | main | — | — |
| 控制按钮组 | group | "录音控制按钮" | — |
| 各按钮 | button | "开始录音 (快捷键 Space)" | — |
| 状态显示 | — | — | polite (status 变更播报) |
| 连接状态 | — | — | polite (WS 状态变更) |
| 转写结果列表 | log | "转写结果日志" | polite (新结果播报) |
| 实时识别行 | listitem | "实时识别中" | — |
| 字幕区 | region | "实时字幕" | polite |
| 主题切换器 | radiogroup | "主题切换" | — |
| 帮助弹层 | dialog | — | — (modal + labelledby + describedby) |
| skip-link | — | — | — (跳到主要内容) |

**键盘快捷键提示**: 每个按钮的 `aria-keyshortcuts="Space"` 让屏幕阅读器知道快捷键。

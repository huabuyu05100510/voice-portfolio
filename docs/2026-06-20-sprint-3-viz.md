## 1. 目标

在 Vosk 实时语音转写 Demo 中加入 **多模态音频可视化区**, 让用户直观看到麦克风信号在 4 个维度的实时状态:

| 维度 | 含义 | 视觉化 | 用途 |
|------|------|--------|------|
| 频谱 | 频率 vs 能量分布 | 热力图 (X=时间, Y=频率, 颜色=能量) | 看清元音/辅音分布, 噪声轮廓 |
| 音高 | 基频 F0 的时间曲线 | 折线图 (X=时间, Y=Hz) | 看语调/语气变化 |
| 能量 | 信号 RMS 强度 | 24 段 VU 条 (绿→黄→红) | 一眼判断说话 vs 静音 |
| VAD | Voice Activity Detection | 圆点指示灯 (绿/灰) | 实时说话检测, 静音时灰色 |

## 2. 架构

```
AudioCaptureEngine (existing)
        │ onAudioData(Int16Array)
        ▼
   handleAudioData (App.tsx)
        │ setLatestAudio(chunk)
        ▼
   <VisualizerPanel stream audioData active />
        │ useEffect([audioData])
        ▼
   AudioVisualizer.updateExternal(chunk)
        │ rAF 60fps tick
        ├─ getByteFrequencyData  → SpectrumRing.pushColumn
        ├─ getFloatTimeDomainData → estimatePitchAutocorrelation → PitchHistory
        ├─ computeRMS            → smoothedVolumeDb
        └─ detectVoiceActivity   → vadIndicator toggle
        ▼
   Canvas 2D 绘制 (不触发 React re-render)
```

### 关键决策

- **Canvas 2D 而非 WebGL**: 4 个 200×200 canvas 用 2D 足够, 不引入着色器复杂度
- **rAF 60fps 自驱**: visualizer 内部 `requestAnimationFrame` 循环, 不依赖 React state 更新
- **AnalyserNode 双模式**:
  - 模式 1 (主): `MediaStream → AnalyserNode.getByteFrequencyData` 真实 FFT
  - 模式 2 (fallback): 外部推 `Int16Array` (无 mic 时用 sample 播放路径)
- **环形缓冲复用**: `SpectrumRing` / `PitchHistory` 零分配, 防止 GC 抖动
- **AnalyserNode FFT 2048, smoothing 0.7** (per spec)

## 3. 算法

### 3.1 RMS (Root Mean Square)

```ts
function computeRMS(samples: Float32Array): number {
  let sum = 0;
  for (let i = 0; i < samples.length; i++) sum += samples[i] * samples[i];
  return Math.sqrt(sum / samples.length);
}
```

### 3.2 VAD (能量阈值法)

```ts
function detectVoiceActivity(samples, threshold = 0.01) {
  return computeRMS(samples) > threshold;
}
```

简单但有效. 阈值 0.01 对应约 -40 dBFS, 排除环境噪声. 加 5 帧 hold 时间防闪烁.

### 3.3 基频估计 (Autocorrelation)

公式:

$$r(\tau) = \sum_{i=0}^{N-\tau} x[i] \cdot x[i+\tau]$$

在 [minLag, maxLag] 范围 (对应 60-500Hz @ 16kHz) 找最强峰, 归一化 `r(τ) / r(0)`, 阈值 > 0.3 才接受.

测试结果: 200Hz 合成信号 → 195-205Hz (±5Hz), 440Hz → 430-450Hz (±10Hz).

> 没上 YIN/MPM: ACF 简单够用, 性能更好 (O(N·L) 而 YIN 是 O(N·L²)).

### 3.4 FFT 频谱降采样

`getByteFrequencyData` 返回 1024 bin (fftSize/2), 视觉化用 128 bin. 用 max-pooling (每 8 bin 取最大), 保留峰值特征.

### 3.5 配色 (热力图)

12 段: 深蓝 → 青 → 绿 → 黄 → 红, 线性插值, 模拟 `viridis`/`inferno` 风格.

## 4. 性能

| 指标 | 目标 | 实测 |
|------|------|------|
| 帧率 | 60 fps | 54-60 fps (headless Chromium) |
| 重渲染开销 | 不触发 React re-render | 0 (canvas 直接绘制) |
| 内存 | 无持续分配 | 环形缓冲 O(1) |
| 启动延迟 | < 100ms | ~50ms (初始化) |

### 4.1 不掉帧的关键

- `setLatestAudio(chunk)` 是 React state 唯一入口, 但视觉化不靠 React re-render
- AudioCapture 每 50-100ms 推一个 chunk (4000-8000 样本), 但 visualizer 是 60fps
- 热力图绘制: `Math.min(bins * 0.5)` 只画 0-4kHz, 砍一半绘制量

### 4.2 可观测性

FPS counter 直接画在 canvas tile header. 截图实测 54 FPS. 低于 30 时会肉眼可见掉帧.

## 7. 已知限制 / 后续

- **无 MediaStream 时**: spectrum canvas 只画 pseudo-bin (音高位置), 频谱无真实形状 — 因为 fallback 不做 FFT. 解决方案: 改用 OfflineAudioContext 重采样
- **多语言基频**: 当前只测了中文/英文, 实际人声 60-500Hz 足够
- **音高不连续**: 静音 → 说话切换时音高会"跳", 可加 median filter 平滑
- **3D 频谱**: 后续可加 3D 旋转 (WebGL 顶点数 < 30 行)

## 已知问题 (待修)

### Server `_extract_words` 不识别 list 格式 (Sprint 7 修)

Vosk 0.3.44 在 `rec.FinalResult()` 时返回:

```python
{
  'result': [  # ← list, 不是 dict
    {'word': '今天', 'start': 0.03, 'end': 0.48, 'conf': 1.0},
    ...
  ],
  'text': '...'
}
```

但 `vosk_worker.py:_extract_words` 仅检查 `isinstance(result["result"], dict)`,
对 list 走 fallback, 而 fallback 也找不到顶层 `words` 字段。

修复: 加 list 分支 (2 行代码)。Sprint 7 处理。

## Task 13.6: Fix stale closure in useRealtimeConversation

### Problem
The auto-connect `useEffect` at line 368-379 had an empty dependency array `[]` with an `eslint-disable-next-line react-hooks/exhaustive-deps` comment. This meant `connect` captured at mount time was stale â€” any changes to `url` or other dependencies would not be reflected if the autoConnect effect ever re-fired (which it wouldn't since deps were empty).

### Changes
1. **`src/hooks/useRealtimeConversation.ts`** â€” Replaced the stale-closure pattern:
   ```typescript
   // Before (stale closure):
   useEffect(() => {
     if (autoConnect) connect();
     return cleanup;
     // eslint-disable-next-line react-hooks/exhaustive-deps
   }, []);

   // After (correct):
   const connectRef = useRef(connect);
   connectRef.current = connect;
   useEffect(() => {
     if (autoConnect) connectRef.current();
     return cleanup;
   }, [autoConnect]);
   ```

2. **`src/__tests__/useRealtimeConversation.test.ts`** â€” Added 3 regression tests:
   - `autoConnect=true` calls connect exactly once on mount
   - `autoConnect=false` does NOT call connect on mount
   - Changing `url` after mount does NOT trigger reconnect (the effect depends on `[autoConnect]`, not `[url]`)

### Net effect
- Removed eslint-disable comment (linter-compliant)
- Stale closure eliminated via ref pattern
- Effect properly depends on `[autoConnect]` so React can control it
